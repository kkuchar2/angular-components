1. Paginated + column chips + cellComponent

Same features as Material app-generic-table: chip column toggle, client-side pagination, sorting, export, built-in uuid/date cells with optional copy, plus reusable cellComponent cells (status badge). Pass [rowActions] for a rightmost ⋮ context menu (app-context-menu). Projected templates still work for one-offs.

2. Row details menu

Same ⋮ trigger with rowMenuVariant="details" — opens a larger panel of read-only fields via [rowDetails] / [rowDetailsTitle] instead of action items.

3. Cell component catalog

Reusable library cells via ColumnDef.cellComponent: PersonCellComponent, MailtoCellComponent, StatusBadgeCellComponent, and BooleanCellComponent.

4. Animated cells

Motion-friendly cells (respect prefers-reduced-motion): pulsing presence, grow-in progress bars, and trend deltas.

5. Server-side pagination

Parent owns the page slice. Export waits for exportRequest.complete().

6. Virtual scroll (10 000 rows)

TanStack Virtual — person + status cells stay cheap enough for large lists. Mutually exclusive with pagination.

7. Empty state

Non-virtual table with no rows.
